FAQ

Frequently Asked Questions

What is an anti-detect browser? ▼

An anti-detect browser creates isolated browsing environments with unique digital fingerprints for each profile. This prevents websites from linking your multiple accounts together through browser fingerprinting techniques like canvas hashing, WebGL vendor strings, audio context analysis, and more.

Is VeilBrowse really free? ▼

Yes. The core browser is MIT licensed and free forever. We don't have a business model yet because we're building the product first. When we do monetize, it will be for optional cloud features — never the local browser. You can create as many profiles as your disk space allows. No profile limits, no hidden tiers, no credit card required.

What proxies are supported? ▼

VeilBrowse supports HTTP, HTTPS, and SOCKS5 proxies with full authentication (username/password). You can add proxies individually or manage them in a pool with built-in health checks that monitor latency and uptime automatically.

How does fingerprint randomization work? ▼

Each profile can have custom CPU cores, RAM amount, GPU vendor/renderer, audio seed, and client rects noise. You can randomize all parameters with one click or set them manually for specific use cases. We maintain a curated pool of realistic hardware configurations based on real market data.

Does it work on Windows and macOS? ▼

Currently VeilBrowse runs natively on Linux with your system's Google Chrome binary. Windows and macOS support is planned. The app is built with Electron + Playwright, so cross-platform support is straightforward. It's just a matter of testing and packaging.
